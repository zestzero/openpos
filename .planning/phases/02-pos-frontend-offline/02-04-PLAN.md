## Research Context (from SUMMARY.md)
- BarcodeDetector API: Hardware-accelerated, available in Chrome/Edge on Android and some desktop. NOT available in Firefox or Safari.
- html5-qrcode: JavaScript library using getUserMedia, works across browsers. Slower than BarcodeDetector but broader support.
- USB keyboard-wedge scanners: Appear as USB keyboards, send characters rapidly (inter-keystroke < 30ms typically). Need pattern detection to distinguish from human typing.

  <action>
    Build keyboard-wedge scanner detection and the scan route:

    1. Create `frontend/src/pos/hooks/useKeyboardWedge.ts`:
       - Keyboard wedge scanners send characters as rapid keystrokes (typically < 30ms between characters, < 50ms before Enter)
       - Algorithm:
         ```
         Track keydown events globally
         Maintain a buffer string and timestamp of last keystroke
         On each keydown:
           - If time since last keystroke < 50ms: append character to buffer
           - If time >= 50ms: start new buffer with this character
           - If key is Enter and buffer length >= 6: emit scanned code, clear buffer
           - Ignore modifier keys (Shift, Ctrl, Alt, Meta)
           - Ignore if target is an input/textarea (don't interfere with typing)
         ```
       - Return:
         ```typescript
         {
           isEnabled: boolean;      // toggle on/off
           lastScan: string | null; // last detected barcode
           toggle: () => void;      // enable/disable
           isScanning: boolean;     // actively receiving keystrokes
         }
         ```
       - Use `useEffect` with `window.addEventListener('keydown', handler)`
       - Clean up on unmount
       - Store scan timeout in ref: if no keystroke for 100ms after Enter, consider scan complete
       - Only process when `isEnabled === true`
       - Minimum barcode length: 4 characters (ignore short accidental sequences)
