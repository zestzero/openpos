## Existing Backend Contracts

From `internal/auth/handler.go`:
- POST `/api/auth/login` → `{ user: User, token: string }`
- POST `/api/auth/login/pin` → `{ user: User, token: string }`
- User type: `{ id: string, email: string, role: string ("owner" | "cashier"), name: string }`
- Auth via `Authorization: Bearer <token>` header

From `internal/catalog/handler.go`:
- GET `/api/catalog/categories` → `{ data: Category[] }`
- GET `/api/catalog/products` → `{ data: Product[] }`
- GET `/api/catalog/variants/search?q=` → `{ data: Variant }`

From `internal/inventory/handler.go`:
- GET `/api/inventory/variants/{id}/stock` → `{ data: StockLevel }`

From `cmd/server/main.go`:
- All `/api/*` routes protected by `AuthMiddleware` (JWT secret from env)
- Server runs on port 8080 default

## Key Decisions from STATE.md
- Single SPA with route separation (POS = mobile-optimized, ERP = desktop-optimized)
- Vite + React (no SSR needed, better PWA support)
- PWA with service workers + IndexedDB
- Product → Variant hierarchy, inventory ledger
- Delta sync for offline
