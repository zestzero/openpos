## Key Backend Context (from Phase 1)

The existing Go backend runs on localhost:8080 (or PORT env) with these endpoints:

- `POST /api/auth/login` ‚Äî email/password ‚Üí {user, token}
- `POST /api/auth/login/pin` ‚Äî cashier PIN login ‚Üí {user, token} (per D-02)
- `GET /api/catalog/categories` ‚Äî list categories
- `GET /api/catalog/products` ‚Äî list products (optional ?category_id= & ?is_active=)
- `GET /api/catalog/products/{id}` ‚Äî get product with variants
- `GET /api/catalog/variants/search?q={barcode_or_sku}` ‚Äî search variant (returns SearchVariantRow)
- `GET /api/inventory/variants/{id}/stock` ‚Äî get stock level

JWT tokens are returned on login with claims: {user_id, email, role}. Sent via `Authorization: Bearer <token>`.

Auth middleware in `internal/middleware/auth.go` verifies tokens and sets context values for user_id, user_email, user_role.

interface Category { id: string; name: string; description: string; }

// GET /api/catalog/products / GET /api/catalog/products/{id}

  category?: Category;
