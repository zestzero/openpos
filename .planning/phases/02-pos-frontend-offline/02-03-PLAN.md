<objective>
Build the POS catalog browsing, cart management, quick keys, and running totals. Cashiers can browse products by category, search by name/SKU, add items to cart, adjust quantities, and see real-time totals in THB.

Purpose: This is the core POS user experience — the "ring up" part of the sale loop. Without this, cashiers cannot add products to a transaction.

Output: Functional catalog grid, cart panel, search, quick keys, and currency formatting — all mobile-optimized and integrated with the Phase 1 backend APIs.
</objective>

### Catalog (already available from Phase 1)
```typescript
// GET /api/catalog/categories → Category[]
interface Category { id: string; name: string; description: string; }

// GET /api/catalog/products?category_id=xxx&is_active=true → ProductWithVariants[]
interface ProductWithVariants {
  product: { id: string; name: string; description: string; category_id?: string; image_url?: string; is_active: boolean; };
  category?: Category;
  variants: Variant[];
}

interface Variant { id: string; product_id: string; sku: string; barcode?: string; name: string; price: number; cost?: number; is_active: boolean; }

// GET /api/catalog/variants/search?q=xxx → SearchVariantRow
interface SearchVariantRow extends Variant { product_name: string; }
```

## Cart types (defined in this plan)

```typescript
// frontend/src/pos/hooks/useCart.ts
interface CartItem {
  variantId: string;
  productName: string;
  variantName: string;
  sku: string;
  price: number;        // satang
  quantity: number;
  subtotal: number;     // satang
}

interface CartState {
  items: CartItem[];
  itemCount: number;
  total: number;        // satang
}

// useCart hook returns:
interface UseCartReturn {
  items: CartItem[];
  itemCount: number;
  total: number;
  addItem: (variant: Variant & { productName: string }) => void;
  removeItem: (variantId: string) => void;
  updateQuantity: (variantId: string, quantity: number) => void;
  clearCart: () => void;
  isEmpty: boolean;
}
```

       ```typescript
       /**
        * Format a satang amount as Thai Baht currency string.
        * @param amount - Amount in satang (100 = 1 THB)
        * @returns Formatted string e.g. "฿150.00"
        */
       export function formatCurrency(amount: number): string {
         return new Intl.NumberFormat('th-TH', {
           style: 'currency',
           currency: 'THB',
         }).format(amount / 100);
       }
       ```

    2. Create `frontend/src/lib/constants.ts`:
       ```typescript
       export const API_BASE_URL = import.meta.env.VITE_API_URL || 'http://localhost:8080';
       ```

    3. Create `frontend/src/pos/hooks/useCart.ts`:
       - Use React `useState` + `useCallback` (local state, not server state — no TanStack Query needed)
       - State shape: `{ items: CartItem[], itemCount: number, total: number }`
       - `addItem(variant)`:
         - If variant already in cart: increment quantity by 1, recalculate subtotal
         - If new: append with quantity=1, subtotal=price
         - Recalculate `itemCount` (sum of all quantities) and `total` (sum of all subtotals)
       - `removeItem(variantId)`: filter out item, recalculate totals
       - `updateQuantity(variantId, quantity)`: if quantity <= 0, remove; else update quantity and subtotal
       - `clearCart()`: reset to empty
       - `isEmpty`: derived boolean
       - Persist cart to `localStorage` under key `openpos_cart` so refresh doesn't lose cart
       - On mount: restore from localStorage if exists
       - IMPORTANT: All amounts stored in satang (integers), formatted on display with `formatCurrency()`

    4. Create `frontend/src/pos/hooks/useFavorites.ts`:
       - Track session-level add-to-cart frequency
       - State: `Record<string, FavoriteItem>` keyed by variantId
       - `recordAdd(variant)`:
         - If already tracked: increment `addCount`
         - If new: add with `addCount: 1`
       - `favorites`: sorted by `addCount` descending, top 8 items
       - Persist to `sessionStorage` under key `openpos_favorites` (session-only, not across browser sessions)
       - On mount: restore from sessionStorage
       - IMPORTANT: Favorites are derived from THIS session's usage only, not server history (per D-04)

    5. Ensure all hooks are typed with TypeScript interfaces and return stable references where appropriate (useMemo for derived values like `isEmpty`, `itemCount`, `total`).
