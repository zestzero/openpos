## Key Decisions from User
- Offline orders finalized without payment (payment deferred to Phase 3)
- Delta sync: send operations (decrement qty), never absolute state
- Idempotency on order UUID: duplicates silently succeed

## Data Model Rules (AGENTS.md)
1. Never flat products — Product → Variant
2. Never quantity column — inventory ledger with derived snapshot
3. Delta sync only — offline sync sends operations, not absolute values
4. Monetary values as integers — store in satang

    Build sqlc queries and the sales service with inventory integration.

    1. Create `db/queries/sales.sql`:
       - `CreateOrder :one` — INSERT INTO orders, return order
       - `CreateOrderItem :one` — INSERT INTO order_items, return item
       - `GetOrderByClientUUID :one` — SELECT by client_uuid (for idempotency check)
       - `GetOrderByID :one` — SELECT full order with items
       - `ListOrders :many` — SELECT with pagination (for future reporting)
       - Use `ON CONFLICT (client_uuid) DO NOTHING` or equivalent for idempotency

    2. Run `sqlc generate` to produce `db/sqlc/sales.sql.go` — verify generated code compiles.

    3. Create `internal/sales/service.go`:
       - Define `Service` struct with fields: `queries *sqlc.Queries`, `inventoryService *inventory.Service`
       - Constructor: `NewService(queries *sqlc.Queries, inventoryService *inventory.Service) *Service`
       - Define `CreateOrderInput` and `OrderItemInput` structs
       - `CreateOrder(ctx context.Context, input CreateOrderInput) (*Order, error)`:
         a. Check idempotency: if `GetOrderByClientUUID` finds existing order, return it immediately (HTTP handler will return 200 instead of 201)
         b. Calculate total_amount: sum of (quantity * unit_price) for all items
         c. Wrap in database transaction (BEGIN/COMMIT/ROLLBACK)
         d. Insert order row
         e. For each item: insert order_item row
         f. For each item: call `inventoryService.DeductStock(ctx, variantID, quantity, orderID, userID)` with reason "SALE"
         g. If any step fails, rollback entire transaction
         h. Return created order
       - `SyncOrders(ctx context.Context, inputs []CreateOrderInput) (SyncResult, error)`:
         a. Iterate over inputs sequentially (NOT parallel — avoids stock race conditions)
         b. For each: call `CreateOrder`
         c. Collect successes and failures
         d. Return `SyncResult` with counts and per-order errors
       - Define `SyncResult` struct: `{ Processed int, Succeeded int, Failed int, Errors []SyncError }`
       - Define `SyncError` struct: `{ ClientUUID string, Error string }`

    4. Important: DeductStock uses the inventory ledger (delta operation). Per AGENTS.md, this is a ledger entry with negative quantity_change, reason "SALE". The inventory service handles this; the sales service just calls it.

    5. Handle edge cases:
       - Zero items → return error
       - Negative quantity → return error
       - Zero/negative unit_price → allow (free items are valid in retail)
       - Stock insufficient → return specific error, rollback transaction
  </action>
