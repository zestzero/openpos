# Phase 4 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

Desktop-first ERP surface: persistent left navigation, top utility bar, table-first content centered on the product/variant grid, right-side drawers for create/edit, dialogs for confirmations, and tabs for reporting.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #ffffff | Background, main surfaces |
| Secondary (30%) | #f5f5f5 | Cards, sidebar, nav, table containers |
| Accent (10%) | #18e299 | Active nav item, selected filters, positive report highlights, import success states |
| Destructive | #d45656 | Archive/delete confirmations and error states |

Accent reserved for: active nav indicator, selected category chip, selected row state, report KPI emphasis, positive trend bars/lines, and success feedback only.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Create product |
| Empty state heading | No products yet |
| Empty state body | Create your first product or import a CSV to start managing variants, stock, and reports. |
| Error state | We couldn’t load ERP data. Check your connection and retry. |
| Destructive confirmation | Archive product/variant: Archive this item? It will stop appearing in POS, but historical sales and stock records stay intact. |
