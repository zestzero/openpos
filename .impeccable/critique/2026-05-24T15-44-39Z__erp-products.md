## Anti-Patterns Verdict

**LLM assessment**: The interface does not feel AI-generated. It uses restrained shadcn patterns, has clear information hierarchy in tables, and does not fall into generic SaaS clichés. However, there are several areas where the design plays it safe to the point of being bland — particularly the flat "Variant 1, Variant 2" labeling and the overly uniform card treatment in ProductTable.

**Deterministic scan**: CLI detector unavailable. Manual structural scan finds:
- Multiple `<Card>` components with near-identical structure (border-border/70, px-6 py-5 header pattern)
- No glassmorphism, gradient text, or side-stripe violations detected
- `animate-fade-in-up` on variant items — animation delay computed from index, which is fine for staggered entry

**Visual overlays**: not available (browser inspection not run; detector unavailable)

## Overall Impression

The foundation is solid — clean table layouts, good use of shadcn primitives, no egregious AI-slop patterns. The core issue is **soulless consistency**: everything looks the same weight, the same gray, the same border treatment. There's no visual personality, no hierarchy that says "this is what matters most right now," and no consideration for power users or accessibility beyond the baseline. The ERP category makes this worse — this is a tool a shop owner lives in daily. It needs to feel fast, confident, and scannable at a glance.

The single biggest opportunity: **dramatically reduce border usage in ProductTable**, add a strong primary action highlight, and give the product header section a distinct visual identity vs. the nested variant rows.

## What's Working

1. **Restock dialog** — well-scoped, good validation with specific error copy, clear call-to-action, physical scene reference ("Positive quantities increase stock").
2. **Empty states** — descriptive, suggest next action, don't just say "No products" with no way forward.
3. **Two-tier table layout** — parent product row + nested variant sub-row is a good pattern for this data model; users can scan variants inline without opening a drawer.
4. **Currency formatting** — `formatTHB` used consistently; stock levels color-coded (amber <10, red = 0) gives quick visual health signal.

## Priority Issues

**[P1] Border noise in ProductTable collapses visual hierarchy**
- **Why it matters**: Every row uses `border-border/70` and `border-border/50` on sub-rows. The result is a grid of equally-weighted lines. The eye has no anchor; the important product name competes with dividers.
- **Fix**: Remove sub-row borders entirely. Use background tint (`bg-muted/20`) + left indentation to indicate nesting. Remove borders from the table container — Card already provides the boundary.
- **Suggested command**: `impeccable layout`

**[P1] "Active in POS" is unexplained jargon**
- **Why it matters**: A shop owner managing products should understand what "active" means without needing to infer. "Active in POS" requires mental translation — does it mean visible? available? both?
- **Fix**: Rename to "Available for sale" or "Show in POS". If there's a separate concept of "archived" for back-office visibility, clarify that distinction inline.
- **Suggested command**: `impeccable clarify`

**[P1] Destructive archive action has no confirmation**
- **Why it matters**: Archive is destructive and irreversible in most implementations. A single misclick archives the wrong product and there's no recovery path.
- **Fix**: Add a confirm dialog before archive, or at minimum a toast with an undo action (e.g., 5-second window). "Archive" is not a safe default-action.
- **Suggested command**: `impeccable harden`

**[P2] No search or filter on the product table**
- **Why it matters**: As products grow, the flat list becomes unusable. A shop with 200+ products requires scrolling or memory to find what they need.
- **Fix**: Add a search input above the table (debounced, searches name + SKU + barcode). Consider a category filter dropdown too.
- **Suggested command**: `impeccable shape`

**[P2] "Variant 1, Variant 2" numbering is a lazy label**
- **Why it matters**: In a product drawer with multiple variants, "Variant 1" tells the user nothing about what's actually different. It's a system label, not a product label.
- **Fix**: Pull from the variant's own name field if set, or auto-generate from SKU/barcode. Fall back to "Variant 1" only when no better name exists.
- **Suggested command**: `impeccable clarify`

**[P2] No keyboard navigation**
- **Why it matters**: All actions require mouse. Power users (Alex) expect keyboard shortcuts for common operations. Modals don't respond to Escape key visibly.
- **Fix**: Add keyboard trap handling for drawers (Esc to close, visible focus ring). Add `aria-label` to icon-only buttons. Consider `Enter` to confirm forms.
- **Suggested command**: `impeccable harden`

**[P2] Variant reordering is click-only with no feedback**
- **Why it matters**: ChevronUp/ChevronDown buttons reorder variants but the `onReorderVariants` handler is a no-op (`async () => undefined`). Users clicking reorder get no visible result.
- **Fix**: Wire up actual reorder API call, or hide the buttons until backend supports it. Don't show UI that pretends to work but doesn't.
- **Suggested command**: `impeccable harden`

## Persona Red Flags

**Alex (Power User)**: No keyboard shortcuts detected. No search/filter on table with 10+ products. Reorder variants UI is non-functional. 3–4 clicks minimum to archive a product. High friction.

**Jordan (First-Timer)**: "Active in POS" and "RESTOCK" are unexplained domain terms. No tooltips or help text in the product drawer. Empty state says "Create your first product" which is good, but the drawer itself offers no guidance for a first-timer. Ambiguous next steps after save.

**Sam (Accessibility-Dependent)**: Variant sub-rows are `<tr>` but the variant cards inside use `<div>` elements — potential table structural issues for screen readers. Icon-only buttons (Archive, Edit) have `aria-label` but the destructive outline button with icon-only variant relies on `aria-label` on the wrapper only. Focus indicators are present (`focus-visible:ring`) but not tested at 200% zoom.

## Minor Observations

- The `controlBaseClass` string in ProductDrawer is a good pattern for consistency — but `rounded-card` is mixed with `rounded-2xl` and `rounded-xl` across the same component. Pick one border radius strategy.
- `min-h-28` on the textarea is oddly specific — `min-h-24` or `min-h-32` would round cleaner.
- The upload zone shows "JPG, PNG, WebP. Best results use a square or near-square image." — that's good descriptive copy. Maintain this standard elsewhere.
- "Or paste image URL" section sits below the drag-drop zone — visual hierarchy suggests URL is secondary, which is correct. But the two-step (drag then URL) isn't obvious until scrolled.
- `archiveBusy` is shared between archive product and archive variant — a single busy state for two different destructive actions could mislead users about what's happening.
