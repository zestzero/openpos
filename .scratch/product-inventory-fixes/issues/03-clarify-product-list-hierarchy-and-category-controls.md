# Clarify Product page list hierarchy and Category controls

## What to build

Improve the ERP Product page list structure so headers are easier to scan, and remove Product Category up/down arrow controls that look interactive but are out of scope. Apply restrained product UI treatment: hierarchy through spacing, weight, subtle surface contrast, and semantic color only where it clarifies state or action.

## Acceptance criteria

- [ ] Product List headers visually stand apart from data rows through stronger typographic hierarchy, spacing, and/or subtle surface treatment.
- [ ] Category headers and rows do not display non-functional up/down sort or reorder arrows.
- [ ] Remaining Category actions are clearly functional and accessible by keyboard and screen reader users.
- [ ] Header styling works on desktop and tablet widths without crowding row content.
- [ ] UI changes avoid decorative side-stripe accents, gradient text, and color-only meaning.
